**1.** O que é o **Node.js**?
Explique sua finalidade e por que ele é considerado um ambiente de execução e não uma linguagem de programação.
O Node.js é um ambiente de execução JavaScript open-source e multiplataforma que permite aos desenvolvedores executar código JavaScript fora de um navegador web. Ele é construído sobre o motor V8 JavaScript do Google Chrome, o que o torna rápido e eficiente. 
---

**2.** Qual a diferença entre **Node.js** e **JavaScript executado no navegador**?
Cite pelo menos **duas diferenças**.
Embora o Node.js e os navegadores utilizem a mesma linguagem (JavaScript) e o mesmo motor de interpretação (V8, no caso do Chrome/Node), eles funcionam em ambientes diferentes e possuem propósitos distintos. 
---

**3.** O que é o **V8 Engine** e qual sua importância para o funcionamento do Node.js?
O V8 Engine é um motor JavaScript e WebAssembly de alto desempenho, de código aberto, desenvolvido pelo Google e escrito em C++. Sua principal função é receber o código JavaScript e executá-lo, traduzindo-o diretamente para código de máquina que o computador entende, em vez de apenas interpretá-lo linha por linha. 
---

**4.** Explique o conceito de **I/O não bloqueante** no Node.js.
Por que isso melhora o desempenho de aplicações?
O I/O (Input/Output - Entrada/Saída) não bloqueante é um dos pilares fundamentais do Node.js e a razão pela qual ele é tão rápido para aplicações em rede. 
---

**5.** O que é o **Event Loop**?
Descreva, de forma resumida, como ele funciona.
O Event Loop (Loop de Eventos) é um mecanismo fundamental no JavaScript que permite que a linguagem realize operações assíncronas e não bloqueantes, mesmo sendo single-threaded (executando apenas uma coisa de cada vez). 
---

**6.** O que são **módulos** no Node.js?
Explique a diferença entre:

* Módulos internos
* Módulos externos
* Módulos criados pelo desenvolvedor
Módulos no Node.js são arquivos ou bibliotecas de código reutilizáveis e independentes, usados para organizar aplicações em partes menores, especializadas e encapsuladas. Eles permitem importar funcionalidades de outros arquivos, nativos ou de terceiros, facilitando a manutenção e escalabilidade do código. 
---

**7.** Para que serve o arquivo **package.json** em um projeto Node.js?
Cite pelo menos **três informações** que ele pode conter.
O arquivo package.json é fundamental em projetos Node.js, servindo como o manifesto e centro de configuração do projeto. Ele armazena metadados essenciais, gerencia dependências (bibliotecas externas) e define scripts de automação. 
---

**8.** O que é o **npm**?
Explique sua função no desenvolvimento de aplicações Node.js.
O npm (Node Package Manager) é o gerenciador de pacotes padrão para o ambiente de tempo de execução Node.js. Ele funciona como uma ferramenta de linha de comando e um imenso registro online de bibliotecas de código JavaScript. 
---

**9.** O que é uma **API REST** e como o Node.js pode ser usado para desenvolvê-la?
Uma API REST (Representational State Transfer) é um conjunto de regras e diretrizes arquiteturais que permite que diferentes sistemas de computador troquem informações de forma segura e eficiente pela internet, geralmente usando o protocolo HTTP.
